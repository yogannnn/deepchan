#  DeepChan Roadmap

Эта дорожная карта содержит **атомарные, точечные задачи** для развития DeepChan.  
Каждый пункт — одно исправление или улучшение. Статусы отмечаются по мере выполнения.

---

## 🔴 КРИТИЧЕСКАЯ БЕЗОПАСНОСТЬ (первоочередное)

- [x] 1. XSS-экранирование комментариев: сначала `html.escape`, потом безопасное восстановление BB-тегов.
- [x] 2. Удаление EXIF из загружаемых изображений (в `save_files` добавить `exif=b''` или очистку).
- [x] 3. Stateless CSRF-защита: HMAC-токены с временной меткой для всех POST-форм.
- [x] 4. Валидация загружаемых файлов через `Image.verify()` перед сохранением.
- [x] 5. Ограничение максимального разрешения изображений (например, `MAX_IMAGE_DIMENSION = 5000`).
- [x] 6. Ограничение размера файла на уровне приложения (уже `MAX_CONTENT_LENGTH = 10MB`). ✅
- [x] 7. SQLite WAL-режим и таймаут: `PRAGMA journal_mode=WAL;`, `connect_args={'timeout': 5}`.
- [x] 8. Запрет загрузки SVG/HTML как изображений (исключить из `ALLOWED_EXTENSIONS`).
- [x] 9. Экранирование имени, темы, сабжа в шаблонах (`|e` или экранирование на Python).
- [ ] 10. Проверка на polyglot файлы: после `verify()` попытаться открыть заново.
