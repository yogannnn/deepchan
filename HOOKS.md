Система хуков (событий) в DeepChan
=====================================

Все плагины взаимодействуют с ядром только через хуки.
Ядро генерирует события, плагины подписываются и получают контекст.

Как подписаться на хук:
------------------------
В функции init_app(app) плагин вызывает app.on('имя_события', callback).

Пример:
def init_app(app):
    def my_handler(**kwargs):
        print("Событие произошло")
    app.on('core.started', my_handler)

Список хуков
-------------

1. core.started
   Когда: один раз при старте приложения.
   Контекст: app (Flask application).
   Для чего: инициализация фоновых потоков, создание таблиц БД.

2. http.before_request
   Когда: перед каждым HTTP-запросом.
   Контекст: request (объект Flask request).
   Для чего: аналитика, блокировка IP, модификация g.

3. http.after_request
   Когда: после каждого HTTP-запроса.
   Контекст: request, response.
   Для чего: добавление заголовков, логирование времени ответа.

4. content.changed
   Когда: создание или удаление поста.
   Контекст: action ('created'/'deleted'), post, thread, board.
   Для чего: уведомления, обновление поискового индекса.

5. post.reported
   Когда: отправка жалобы на пост.
   Контекст: post, report.
   Для чего: оповещение модераторов, автоматическое скрытие.

6. ui.before_render
   Когда: перед рендерингом любого шаблона.
   Контекст: template_name (имя шаблона), context (словарь переменных).
   Особенность: можно изменять context, добавлять/удалять переменные.
   Для чего: скрытие досок/тредов, фильтрация контента.

7. ui.header_rendering
   Когда: при рендеринге секции header.
   Контекст: request.
   Возврат: HTML-строка для вставки в шапку.
   Для чего: навигационные ссылки, статус-бары.

8. ui.footer_rendering
   Когда: при рендеринге секции footer.
   Контекст: request.
   Возврат: HTML-строка для вставки в подвал.
   Для чего: цитаты, счётчики, копирайты.

9. board.opening
   Когда: при открытии страницы доски (/b/).
   Контекст: board.
   Для чего: логирование просмотров, контроль доступа.

10. thread.opening
    Когда: при открытии страницы треда (/b/thread/1).
    Контекст: thread, board.
    Для чего: аналитика, отметки "прочитано".

11. admin.setting_changed
    Когда: после сохранения настроек в админке.
    Контекст: changed_keys (список ключей).
    Для чего: сброс кеша, логирование изменений.

12. admin.action
    Когда: при административных действиях (бан, фильтр, удаление).
    Контекст: action (строка типа 'ban_added'), плюс специфичные параметры (ip, reason, pattern).
    Для чего: аудит, интеграция с внешними системами.


Приоритеты плагинов
-------------------
Плагины могут указывать параметр "priority" в plugin.json (целое число, по умолчанию 50).
Чем МЕНЬШЕ число, тем ВЫШЕ приоритет.
Плагины с более высоким приоритетом загружаются раньше, их колбэки вызываются первыми, и их виджеты отображаются выше/левее.

Хуки сервисного слоя (появились в v1.2.9)
-----------------------------------------

posts.before_create(board, thread, form, ip_address)
  Вызывается перед созданием поста. Позволяет плагинам отклонить пост (abort) или модифицировать данные формы.

posts.after_create(post, board, thread)
  Вызывается сразу после успешного создания поста. Идеально для уведомлений, логирования, аналитики.

posts.before_render(post)
  Вызывается перед рендерингом поста. Позволяет плагинам изменить содержимое поста (например, применить фильтры).

threads.before_render(thread)
  Вызывается перед рендерингом треда. Позволяет модифицировать список постов или добавить информацию в шапку.

threads.list_loaded(threads, board_id)
  Вызывается после загрузки списка тредов доски. Позволяет фильтровать или сортировать треды.

media.before_process(file, post, board, thread)
  Вызывается перед сохранением файла. Позволяет отклонить файл (антивирус, NSFW-сканер).

media.after_process(file, post, board, thread)
  Вызывается после сохранения файла. Позволяет наложить водяной знак, проанализировать файл, отправить уведомление.
