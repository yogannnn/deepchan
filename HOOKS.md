Система хуков (событий) в DeepChan

Все плагины взаимодействуют с ядром только через хуки.
Ядро генерирует события, плагины подписываются и получают контекст.

Как подписаться на хук

def init_app(app):
    def my_handler(**kwargs):
        pass
    app.on('имя_события', my_handler)

Список хуков

core.started
Вызывается один раз при старте приложения, после загрузки всех плагинов.
Идеально для инициализации фоновых потоков или таблиц БД.
Контекст: app — экземпляр Flask приложения.

http.before_request
Вызывается перед обработкой любого HTTP-запроса.
Контекст: request — объект запроса Flask.

http.after_request
Вызывается после обработки любого HTTP-запроса.
Контекст: request, response.

content.changed
Вызывается при создании или удалении поста.
Контекст:
- action: 'created' или 'deleted'
- post: объект модели Post
- thread: объект модели Thread
- board: объект модели Board

admin.setting_changed
Вызывается при сохранении настроек в админке.
Контекст: changed_keys — список изменённых ключей.

admin.action
Вызывается при административных действиях (бан, фильтр и т.д.).
Контекст:
- action: 'ban_added', 'filter_added' и др.
- Специфичные параметры (ip, reason, pattern и т.п.)

ui.header_rendering и ui.footer_rendering
Вызываются при рендеринге каждой страницы. Плагин может вернуть HTML-строку, которая будет вставлена в шапку или подвал сайта.
Контекст: request
Ожидаемый возврат: строка с HTML (будет помещена в header или footer).
